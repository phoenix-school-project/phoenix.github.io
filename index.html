1. 프로젝트 개요

- 프로젝트 주제 및 선정 배경

1. To Do list는 많은 사람들이 일상생활에서 사용하는 도구 중 하나이므로 개발을 했을 때 많은 사용자들이 실제로 이용할 가능성이 높습니다.
2. 단순한 것처럼 보이지만, 사용자 인터페이스, 데이터베이스 관리, 로그인 및 인증, 실시간 알림 등 다양한 기술적인 요소를 포함할 수 있습니다.
3. 다양한 기술을 연습하고 익힐 수 있는 기회를 제공합니다.
4. Microsoft To Do 앱을 모티브로 사용하면서 불편했던 점이나 추가되었으면 하는 기능을 더한 앱을 만들어 보고 싶다는 이유가 있습니다.
5. 실사용이 가능한 웹 애플리케이션을 직접 개발해 보면서 기획부터 디자인, 제작(개발), 배포 하기까지의 일련의 과정들을 경험해 볼 수 있습니다.
6. 자신의 기술적인 역량을 향상시킬 수 있다는 점이 있습니다.

- 개발 및 수행 목표

주기능

- 작업 목록 공유 기능
- 작업 목록 관리 기능 (작업 목록 삭제, 추가, 작업 목록 이름 변경)
- To do list의 기본적인 기능 (작업 삭제, 등록, 수정, 기한 설정 )
- 작업 반복 기능 (자동으로 작업을 설정한 시간 기준으로 자동으로 작업 목록에 추가)
- 캘린더 기능 (완료한 작업과 완료하지 못한 작업 등 지금까지 한 작업들이 언제 완료 되었고 언제 까지 완료해야 하는 지를 캘린더에 표시하고 한 눈에 확인 할 수 있음) *추가되었으면 하는 기능*

부수기능

- 작업 세분화 기능
- #해시태그 검색 기능
- 테마 변경 기능

페이지 구성 계획

1. 로그인&로그아웃 페이지
2. 회원가입 페이지
3. 작업 목록 관리 & 작업 목록 페이지
4. 목록에 작업 등록 페이지
5. 검색 페이지
6. 캘린더 페이지
7. 작업 공유 페이지
